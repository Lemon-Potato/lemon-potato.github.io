Vue Prop

Created2022-01-10|Updated2022-11-01
|Post View:

背景

  • 结合 Vue 官方文档，将 Prop 的关键点进行梳理与记录

Prop 命名

  • Prop 命名可以使用 camelCase(驼峰命名法)，但是在 HTML 中，需要使用其等价的 kebab-case(短横线分隔命名)命名

常用属性 (Prop验证发挥关键作用)

type (类型)

String 字符串

Number 数字

Boolean 布尔

Array 数组

Object 对象

Function 函数

Date 日期

Symbol 独一无二的值(es6) // or any other constructor

default (默认值)

基础数据类型：直接赋值

对象数据类型：可以用函数赋值 ()=>{}

required (必填项)
  • 默认为 false
  • 为 true 时，说明父级必须传入，否则会报错
validator (校验)
  • 传入校验规则，验证传入的值是否符合规定

Prop 验证

  • 可以为组件的 prop 指定验证要求，这在开发一个会被别人用到的组件时尤其有帮助
  • 为了定制 prop 的验证方式，你可以为 props 中的值提供一个带有验证需求的对象

Prop 是一种单向数据流

  • 所有的 prop 都使得其父子 prop 之间形成了一个单向下行绑定。意味着不应该在一个子组件内部改变 prop。如果你这样做了，Vue 会在浏览器的控制台中发出警告
常见的试图变更 prop 的方式
  • 使用 prop 来传递一个初始值，这个子组件接下来希望将初始值作为一个本地的 prop 数据来使用。这个情况下，最好定义一个本地的 data property 并将这个 prop 用作其初始值
  • 这个 prop 以一种原始的值传入且需要进行转换。在这种情况下，最好使用这个 prop 的值来定义一个计算属性

注意在 JavaScript 中对象和数组是通过引用传入的，所以对于一个数组或对象类型的 prop 来说，在子组件中改变变更这个对象或数组本身将会影响到父组件的状态
